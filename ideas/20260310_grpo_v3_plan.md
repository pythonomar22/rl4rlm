# GRPO v3 Plan

## What Changes from v2

### Task Mix (mixed_v3)
v2 mix: 40% NIAH, 20% Multi-NIAH, 20% Doc-Classify, 10% DFQA, 10% CodeDebug

v3 proposed mix: 30% NIAH, 15% Multi-NIAH, 15% Doc-Classify, 15% Multi-Hop QA, 10% DFQA, 10% CodeDebug, 5% Hard NIAH (100K+ docs)

Rationale:
- Multi-Hop QA is the most compelling benchmark for showing RLM value (base: 50%, 0% at >50K)
- NIAH can be reduced slightly since base model is already good (81%)
- Adding Hard NIAH (100K+ char docs) would test long-context generalization
- Keep CodeDebug and DFQA for breadth

### Hyperparameters
- Same LR (2e-6 base, ~20e-6 effective)
- Same temperature (1.0)
- Same K=8
- Potentially increase batch_size to 4 for better gradient stability
- Resume from v2's best checkpoint (step 10 or wherever eval is best)

### Multi-Hop QA Scoring
Already implemented in rl_tinker.py:
- score_multi_hop returns {score: 1.0/0.5/0.0, partial: bool}
- Contains match → 1.0
- Key term 70%+ overlap → 0.5

### Monitoring
- Log model_id at startup (already added)
- Track per-task-type reward averages
- Watch for NIAH regression with reduced weight
- Monitor multi-hop QA improvement specifically

## Decision Criteria
- Wait for v2 evaluation results
- If v2 step 5 shows NIAH regression, increase NIAH weight in v3
- If doc-classify plateaus, reduce its weight
- If multi-hop QA improves with training, that's the key paper result

## External Benchmarks Before v3
- Run OOLONG baseline (document classification from long context)
- Run RULER baseline (synthetic long-context tasks)
- These establish external credibility for the paper
